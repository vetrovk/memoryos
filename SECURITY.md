# Security Policy

## Supported Versions

MemoryOS is a public beta. Security fixes are generally applied to the latest published version. When practical, reproduce an issue against the latest release or the current `main` branch. Prerelease compatibility may change before v1.0.

## Reporting a Vulnerability

Do not publish vulnerability details in a public GitHub issue. Use [Report a vulnerability](https://github.com/vetrovk/memoryos/security/advisories/new) in this repository's Security Advisories area. GitHub Private Vulnerability Reporting sends the report privately to the repository maintainers.

Please include:

- the affected MemoryOS version or commit;
- operating system and Python version;
- the affected command or API path;
- a concise impact description;
- reproducible steps or a minimal proof of concept; and
- any safe suggested remediation.

Do not include real MemoryOS notes, private memory-home contents, access tokens, API keys, or destructive proof-of-concept steps.

## Response and Disclosure

The maintainer will review reports and may request more information. Coordinate disclosure with the maintainer before making details public. MemoryOS does not offer a guaranteed response or fix timeline, bounty, CVE, or reward. Reports outside the MemoryOS scope may be closed or redirected.

## Scope

This policy covers vulnerabilities in MemoryOS itself. Vulnerabilities or privacy behavior in GitHub CLI, Codex, Python, the operating system, filesystem, editor, or other external tools should be reported to the relevant project or vendor.
