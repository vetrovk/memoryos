# Security Policy

## Supported Versions

Security fixes are made on the current `main` branch during the public beta.

## Reporting A Vulnerability

Please do not publish sensitive vulnerability details in a public issue. Use the repository owner's GitHub profile to request a private reporting channel, and include a minimal reproduction, affected version or commit, impact, and any safe mitigation you found.

Do not include personal MemoryOS data, access tokens, API keys, or real memory notes in the report.
